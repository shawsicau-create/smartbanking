\subsection{MiKTeX（Windows）}

MiKTeX是Windows上最推荐的LaTeX发行版，支持按需自动安装宏包。

\begin{enumerate}[leftmargin=2em]
  \item 访问\url{https://miktex.org/download}，下载Basic MiKTeX Installer（约230 MB）
  \item 运行安装程序，选择"Install for current user"（无需管理员权限）
  \item 安装路径默认为\texttt{\%LOCALAPPDATA\%\textbackslash Programs\textbackslash MiKTeX\textbackslash}
  \item 打开MiKTeX Console → Settings →"Install missing packages"设为Always
  \item Updates →"Check for updates"更新一次
\end{enumerate}

\section{MCP服务器配置详解}
% ------------------------------------------------------------

MCP（Model Context Protocol）让AI助手能够访问外部数据源和工具。Qoder的MCP配置文件位于：

\begin{lstlisting}
# Windows
%APPDATA%\Qoder\SharedClientCache\extension\local\mcp.json

# macOS
~/Library/Application Support/Qoder/SharedClientCache/extension/local/mcp.json
\end{lstlisting}

\subsection{推荐配置的MCP服务器}

\begin{table}[H]
\centering
\begin{tabular}{lll}
\toprule
\textbf{服务名} & \textbf{启动命令} & \textbf{用途} \\
\midrule
fetch & \texttt{uvx mcp-server-fetch} & HTTP网页抓取 \\
playwright & \texttt{npx @playwright/mcp@latest} & 浏览器自动化 \\
excel & \texttt{npx @negokaz/excel-mcp-server} & Excel读写 \\
ppt & \texttt{uvx ppt\_mcp\_server} & PPT编辑 \\
word & \texttt{uvx word\_mcp\_server} & Word文档 \\
stata-mcp & \texttt{uvx stata-mcp} & Stata统计分析 \\
\bottomrule
\end{tabular}
\end{table}

\subsection{完整mcp.json参考配置}

\begin{lstlisting}[style=json]

    "fetch": {

    "playwright": {
      "command": "npx",
      "args": ["@playwright/mcp@latest"]
    },
    "excel": {
      "command": "npx",

      "command": "uvx",

\subsection{IDE安装/登录问题}

\begin{table}[H]

\begin{tabular}{lp{3.5cm}p{5.5cm}}
\toprule
\textbf{现象} & \textbf{原因} & \textbf{解决} \\

\end{tabular}

\subsection{Python/Node环境问题}

\begin{itemize}[leftmargin=2em]
  \item \textbf{多版本Python冲突}：优先Anaconda；\texttt{where python}看路径，环境变量上移Anaconda
  \item \textbf{pip install SSL错误}：\texttt{pip config set global.index-url https://pypi.tuna.tsinghua.edu.cn/simple}
  \item \textbf{npm install慢}：\texttt{npm config set registry https://registry.npmmirror.com}
  \item \textbf{TensorFlow装不上}：业绩预测有SARIMA即可得分，LSTM可选
\end{itemize}

\subsection{MCP类问题}

\begin{itemize}[leftmargin=2em]
  \item \textbf{找不到npx/uvx}：Node.js与uv都得装，且加入PATH。\texttt{pip install uv}装uv
  \item \textbf{mcp.json改完没生效}：必须\textbf{完全退出IDE}再重启（任务栏右键退出，不只是关窗口）
  \item \textbf{playwright报EPERM}：不要装在\texttt{C:\textbackslash Program Files\textbackslash}下；改用户目录
  \item \textbf{stata-mcp找不到Stata}：在env.STATA\_PATH填本机Stata实际路径
  \item \textbf{word/ppt MCP启动慢}：首次uvx需联网下载，等1--3分钟
\end{itemize}

\subsection{Skill类问题}

\begin{itemize}[leftmargin=2em]
  \item \textbf{AI不识别Skill}：①升级IDE；②使用skill finder重新安装；③检查description是否含明确触发词
  \item \textbf{自写Skill不激活}：description写得太宽泛，AI不知何时调用——把场景写具体（如"当用户提到X或Y时使用"）
\end{itemize}

\section{与stata-mcp联动配置}
% ------------------------------------------------------------

stata-mcp有两种方案，分别对应不同的配置方式：

\subsection{方案一：SepineTam/mcp-for-stata（独立Server，本课程默认）}

本课程mcp.json使用的是SepineTam方案，它是独立的MCP Server，\textbf{无需IDE扩展}，直接通过subprocess调用Stata CLI：

\begin{lstlisting}[style=json]
"stata-mcp": {
  "command": "uvx",
  "args": ["stata-mcp"],
  "env": {
    "STATA_PATH": "C:\\Program Files\\Stata18\\StataMP-64.exe",
    "MCP_STATA_LOGLEVEL": "INFO"
  }
}
\end{lstlisting}

\textbf{环境检查}（验证配置是否正确）：

\begin{lstlisting}[style=shell]
# 检查 stata-mcp 是否能找到 Stata
uvx stata-mcp doctor
\end{lstlisting}

正常输出示例：
\begin{lstlisting}
stata-mcp v1.17.0 -- Doctor Report
[PASS] os: macOS (Darwin 25.3.0, arm64)
[PASS] python: 3.13.5
[PASS] uv: uv 0.11.13
[PASS] stata_cli: /usr/local/bin/stata-mp
[PASS] stata_execution: OK (0.1s)
[PASS] guard: enabled, loaded 27 rules
Summary: 12 passed, 0 failed
\end{lstlisting}

\begin{notebox}[路径注意]
如安装的是SE版本，把StataMP-64.exe改为StataSE-64.exe；如安装路径不同，整段绝对路径也要相应修改。\textbf{注意JSON中反斜杠必须写成\textbackslash\textbackslash}。
\end{notebox}

\subsection{方案二：hanlulong/stata-mcp（IDE扩展方案）}

如果使用Qoder/VS Code/Cursor等支持扩展的IDE，可以选择安装\texttt{DeepEcon.stata-mcp}扩展。此方案的架构如下：

\begin{verbatim}
┌─────────────────┐        HTTP         ┌──────────────────────┐
│  IDE (客户端)    │ ──mcp-remote──────▶ │  Stata MCP Server    │
│  mcp.json 配置   │                     │  localhost:4000      │
└─────────────────┘                     │  (由IDE扩展提供)      │
                                        └──────────────────────┘
\end{verbatim}

\textbf{Step 1：安装DeepEcon.stata-mcp扩展}（\textbf{关键步骤，不可跳过}）

\begin{lstlisting}[style=shell]
# VS Code
code --install-extension DeepEcon.stata-mcp

# Cursor
cursor --install-extension DeepEcon.stata-mcp

# Qoder / Antigravity
# 在扩展市场搜索 "Stata MCP" 安装
\end{lstlisting}

或在IDE中：扩展视图（Ctrl+Shift+X / Cmd+Shift+X）→搜索"Stata MCP"→安装。

\begin{warningbox}[重要提醒]
此方案\textbf{必须先安装扩展}！扩展启动后才会在localhost:4000监听，否则mcp-remote无法连接，会报"connection refused"错误。安装成功后状态栏应显示"Stata"。
\end{warningbox}

\textbf{Step 2：配置mcp.json}（仅Qoder/Claude Desktop需要此配置）

\begin{lstlisting}[style=json]
"stata-mcp": {
  "command": "npx",
  "args": ["-y", "mcp-remote", "http://localhost:4000/mcp-streamable"]
}
\end{lstlisting}

\textbf{Step 3：验证服务}

\begin{lstlisting}[style=shell]
# 健康检查（扩展启动后执行）
curl -s http://localhost:4000/health
# 期望返回: {"status":"ok","service":"Stata MCP Server","version":"0.4.1","stata_available":true}
\end{lstlisting}

\midrule
