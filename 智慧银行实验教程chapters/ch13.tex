\section{CLI工具概述}
% ============================================================

命令行界面（CLI）是AI辅助开发的核心执行层。与图形界面相比，CLI工具具有以下优势：

\begin{itemize}
  \item \textbf{自动化友好}：可编写脚本批量执行，适合CI/CD流水线
  \item \textbf{资源占用低}：无需图形界面，服务器环境也能运行
  \item \textbf{组合能力强}：通过管道（pipe）将多个工具串联
  \item \textbf{AI原生}：大语言模型天然擅长生成和执行命令
\end{itemize}

本章将系统介绍课程中涉及的所有CLI工具，帮助你构建完整的AI开发工具链。

\section{CNB CLI——代码仓库管理}
% ============================================================

CNB（Cloud Native Build）是课程使用的代码托管平台，提供\texttt{cnb}命令行工具管理仓库、组织、Issue等资源。

\subsection{安装}

\begin{lstlisting}[style=shell]
# macOS / Linux
npm install -g @anthropic-ai/cnb

# 验证安装
cnb --version
\end{lstlisting}

\subsection{登录与认证}

\begin{lstlisting}[style=shell]
# 登录CNB（OAuth2授权流程）
cnb login

# 终端会显示授权链接和user_code，浏览器打开后确认授权
# 登录成功后，凭证自动保存，后续命令无需重复登录

# 检查登录状态
cnb status
\end{lstlisting}

\subsection{常用命令}

\begin{lstlisting}[style=shell]
# 查看仓库信息
cnb repositories get-repos

# 克隆仓库
git clone https://cnb.cool/组织名/仓库名.git

# 创建Issue
cnb issues create-issue --repo 组织名/仓库名 --title "Issue标题"

# 查看Pull Request
cnb pulls list-pulls --repo 组织名/仓库名
\end{lstlisting}

\subsection{Git集成}

CNB CLI可作为Git凭证助手，自动管理HTTPS认证：

\begin{lstlisting}[style=shell]
# 配置Git使用CNB凭证助手
git config --global credential.https://cnb.cool.helper "cnb git-credential"

# 之后git push/pull会自动使用CNB凭证
git push origin main
\end{lstlisting}

\section{Skills CLI——AI能力包管理}
% ============================================================

Skills是领域知识文件（SKILL.md），赋予AI专业能力。Skills CLI用于安装和管理这些能力包。

\subsection{安装}

\begin{lstlisting}[style=shell]
# 通过npx直接使用（无需全局安装）
npx skills add 作者/技能名

# 全局安装Skills CLI
npm install -g @anthropic-ai/skills
\end{lstlisting}

\subsection{常用操作}

\begin{lstlisting}[style=shell]
# 安装Skill
npx skills add LearnPrompt/luban-skill

# 安装到全局（所有项目可用）
npx skills add LearnPrompt/luban-skill -g

# 列出已安装的Skill
npx skills list

# 搜索Skill
npx skills search "金融分析"
\end{lstlisting}

\section{AI CLI工具}
% ============================================================

AI CLI工具是直接与大语言模型交互的命令行客户端，是AI辅助开发的核心执行工具。

\subsection{Claude Code}

Claude Code是Anthropic官方的AI编程助手，提供命令行交互方式。

\begin{lstlisting}[style=shell]
# 安装
npm install -g @anthropic-ai/claude-code

# 启动交互式会话
claude

# 执行单条命令
claude -p "解释这段代码的作用"

# 在管道中使用
cat main.py | claude -p "找出潜在的bug"
\end{lstlisting}

\textbf{核心特性}：上下文感知、多文件编辑、Git集成、MCP支持。

\subsection{Codex CLI}

Codex CLI是OpenAI的命令行AI助手，基于GPT-4o模型。

\begin{lstlisting}[style=shell]
# 安装
npm install -g @openai/codex

# 启动
codex

# 执行任务
codex "创建一个Python脚本，分析CSV数据并生成图表"
\end{lstlisting}

\subsection{Qoder CLI}

Qoder CLI支持多模型切换，适合需要灵活选择模型的场景。

\begin{lstlisting}[style=shell]
# 安装
npm install -g @qoder-ai/qodercli

# 启动
qoder

# 切换模型
qoder --model gpt-4o
qoder --model claude-sonnet
\end{lstlisting}

\subsection{AI CLI工具对比}

\begin{table}[H]
\centering
\begin{tabular}{llll}
\toprule
\textbf{工具} & \textbf{厂商} & \textbf{核心模型} & \textbf{特点} \\
\midrule
Claude Code & Anthropic & Claude Sonnet & 上下文长、代码理解强 \\
Codex CLI & OpenAI & GPT-4o & 多模态、推理能力强 \\
Qoder CLI & Qoder AI & 多模型切换 & 灵活、支持自定义 \\
\bottomrule
\end{tabular}
\end{table}

\section{MiMo CLI——小米AI编程助手}
% ============================================================

MiMo CLI是小米推出的AI编程助手，提供命令行交互方式，支持代码生成、解释、重构等功能。

\subsection{安装}

\begin{lstlisting}[style=shell]
# macOS / Linux（推荐）
curl -fsSL https://mimo.xiaomi.com/install | bash

# Windows（推荐使用npm）
npm install -g @mimo-ai/cli

# 验证安装
mimo --version
\end{lstlisting}

\subsection{启动与使用}

\begin{lstlisting}[style=shell]
# 启动交互式会话
mimo

# 直接提问
mimo "如何用Python读取Excel文件？"

# 代码解释
mimo explain main.py

# 代码重构
mimo refactor main.py --style clean
\end{lstlisting}

\subsection{最佳实践}

\begin{itemize}
  \item \textbf{终端选择}：Mac用户推荐在iTerm或VS Code终端中使用，体验更佳
  \item \textbf{上下文管理}：MiMo CLI会自动维护会话上下文，支持多轮对话
  \item \textbf{模型选择}：支持多种模型，可根据任务类型选择最适合的模型
\end{itemize}

\section{CC Switch——多账号管理}
% ============================================================

CC Switch是AI CLI多账号/多服务商管理工具，提供可视化界面统一管理Claude Code、Codex、Gemini CLI等7款工具的配置。

\subsection{安装}

\begin{lstlisting}[style=shell]
# macOS
brew install --cask cc-switch

# Windows：从GitHub Releases下载.msi安装
# https://github.com/farion1231/cc-switch/releases
\end{lstlisting}

\subsection{核心功能}

\begin{itemize}
  \item \textbf{多服务商切换}：50+预置服务商一键切换
  \item \textbf{系统托盘快切}：快速切换不同AI CLI配置
  \item \textbf{统一MCP管理}：集中管理所有MCP服务器配置
  \item \textbf{用量追踪}：监控API调用次数和费用
  \item \textbf{云端同步}：配置云端备份，多设备同步
\end{itemize}

\subsection{使用场景}

当你需要在不同AI服务商之间切换时（例如Claude额度用完切换到GPT-4o），CC Switch可以一键切换，无需手动修改配置文件。

\section{OpenCLI与OpenClaw}
% ============================================================

\subsection{OpenCLI——把网站变成命令行}

OpenCLI可以将任何网站转换为命令行接口，实现网页操作的自动化。

\begin{lstlisting}[style=shell]
# 安装
npm install -g @jackwener/opencli

# 验证安装
opencli --version

# 配置Chrome扩展连接
opencli setup
\end{lstlisting}

\subsection{OpenClaw——AI Skill包管理}

OpenClaw是另一个Skill包管理器，提供丰富的社区Skill资源。

\begin{lstlisting}[style=shell]
# 安装
npm install -g openclaw

# 验证安装
openclaw --version

# 安装Skill
openclaw install academic-paper-analysis
openclaw install arxiv
\end{lstlisting}

\section{MCP/Skill/CLI三种范式对比}
% ============================================================

在AI辅助开发生态中，MCP、Skill、CLI是三种核心范式，各有侧重：

\begin{table}[H]
\centering
\begin{tabular}{llll}
\toprule
\textbf{维度} & \textbf{MCP} & \textbf{Skill} & \textbf{CLI} \\
\midrule
本质 & 连接协议 & 领域知识文件 & Shell命令 \\
类比 & USB-C接口 & 操作手册 & 万能遥控器 \\
开发成本 & 高（写Server） & 低（写Markdown） & 零 \\
适合场景 & 企业SaaS连接 & 知识沉淀复用 & 本地自动化 \\
\bottomrule
\end{tabular}
\end{table}

\begin{notebox}[黄金组合]
CLI（执行层）+ Skill（知识层）+ MCP（连接层）= 完整AI工具链。

日常以CLI + Skill为主，MCP在需要跨系统数据连接时介入。例如：
\begin{itemize}
  \item 使用\texttt{mimo}（CLI）生成代码
  \item 参考\texttt{finance-expert}（Skill）提供领域知识
  通过\texttt{stata-mcp}（MCP）连接Stata执行统计分析
\end{itemize}
\end{notebox}
