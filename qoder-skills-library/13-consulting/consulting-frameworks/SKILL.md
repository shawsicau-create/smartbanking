# 咨询分析框架

## Purpose

提供经典咨询分析框架工具箱，帮助用户运用 McKinsey 7S、BCG矩阵、波特五力模型、SWOT分析等框架系统化地分析商业战略问题。适用于管理学案例研究、企业诊断、战略规划等场景。

## When to Use

- 需要系统化分析企业战略问题时使用
- 管理学案例研究写作时需要选择合适的分析框架
- 企业诊断、组织变革规划需要结构化思维时触发
- 触发词："咨询框架"、"McKinsey"、"BCG"、"波特五力"、"SWOT"、"7S模型"

## Instructions

### Step 1: 确定分析场景

询问用户：
- 分析对象是什么？(企业/行业/产品/组织)
- 分析目标是什么？(战略定位/竞争优势/组织诊断)
- 需要哪个框架？或由技能根据场景推荐

### Step 2: 选择并应用框架

根据场景推荐框架：

| 框架 | 适用场景 |
|------|----------|
| McKinsey 7S | 组织变革、内部诊断 |
| BCG矩阵 | 产品组合管理、投资优先级 |
| 波特五力 | 行业竞争分析、进入决策 |
| SWOT | 通用战略定位、快速评估 |
| 价值链 | 运营效率、成本优化 |
| 安索夫矩阵 | 增长战略选择 |

### Step 3: 结构化输出

按框架维度逐项分析：
- 每个维度的事实证据
- 维度间的关联与矛盾
- 核心发现与战略建议

### Step 4: 生成报告

输出结构化分析报告，包含：
- 框架选择依据
- 各维度分析详情
- 关键洞察与建议
- 后续行动方案

## Example Prompts

- "用McKinsey 7S框架分析这家公司的组织问题"
- "用波特五力模型评估新能源汽车行业的竞争格局"
- "BCG矩阵帮我分析这个产品组合的投资优先级"

## Requirements

### Knowledge
- McKinsey 7S模型、BCG矩阵、波特五力、SWOT、价值链、安索夫矩阵等经典框架

## Best Practices

1. **框架匹配场景** — 不强行套用，先诊断再选框架
2. **证据驱动** — 每个维度结论需有事实支撑，不空泛
3. **维度交叉分析** — 识别框架维度间的关联和矛盾

## Common Pitfalls

- ❌ 机械套用框架而不适配具体场景
- ❌ 分析停留在描述层面，缺乏诊断性洞察
- ❌ 忽略框架维度间的动态关联
